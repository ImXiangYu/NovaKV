# 为什么需要日志系统
- 输出分级：把信息按 Trace/Debug/Info/Warn/Error/Fatal 分层，避免所有输出混在一起。
- 可控开关：按运行环境调节日志等级，减少噪声。
- 定位问题：自动携带时间戳与文件/行号，便于定位问题来源。
- 输出统一：比零散的 std::cout/std::cerr 更稳定，后续便于扩展为文件日志。

# 实现原理（基础版）
- 维护一个全局等级阈值（g_level），Log() 先比较等级再决定是否输出。
- Log() 生成统一格式：时间戳 + 等级 + 文件:行号 + 消息。
- 通过宏封装 Logger::Log(...)，自动填充 __FILE__/__LINE__，调用更简洁。
- 可选文件输出：设置路径后写入文件，否则写到控制台。

# 当前实现的边界
- 同步输出：每次 Log() 直接写入（易理解，但高并发下性能一般）。
- 无滚动与异步：暂不做按大小/时间切割，也不做后台写线程。
